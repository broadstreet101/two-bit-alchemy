# Publication Readiness Checklist

This checklist is the reusable pre-publication review for every Two-Bit Alchemy shelf, exhibit, Field Note, artifact, project, or related content group.

It is documentation only. It does not modify the website, theme, templates, CSS, WordPress configuration, or live content.

Use this checklist after Workbench material has been drafted and before anything is treated as public-ready.

## Repository Content Complete

- [ ] Final content exists in the repository or approved WordPress content location.
- [ ] Workbench notes remain preserved as source material.
- [ ] Public copy no longer depends on raw Workbench-only placeholders.
- [ ] Unknown details are either resolved, removed, or clearly marked uncertain.
- [ ] No dates, names, locations, motivations, relationships, or story details have been invented.

## Editorial Review Complete

- [ ] The piece has a clear purpose.
- [ ] The voice sounds like Two-Bit Alchemy, not marketing copy.
- [ ] The writing preserves Dada's voice rather than replacing it.
- [ ] The piece explains why the subject matters.
- [ ] The piece teaches, preserves, clarifies, or rewards curiosity.
- [ ] Any private material has been intentionally kept private or approved for publication.

## Fact / Uncertainty Review

- [ ] Known facts are separated from uncertain memories or assumptions.
- [ ] Unverified details are labeled honestly.
- [ ] Names, dates, places, and relationships have been checked where possible.
- [ ] Sensitive personal, medical, family, or third-party details have been reviewed.
- [ ] Copyright-sensitive or permission-sensitive material has been reviewed.

## Image Inventory Complete

- [ ] Required photographs have been identified.
- [ ] Original image files are preserved separately from web-ready exports.
- [ ] Selected images function as evidence, not decoration.
- [ ] Images are close enough, clear enough, and contextual enough to support the story.
- [ ] Image filenames are descriptive.
- [ ] Credits, source notes, and permission notes are recorded where needed.

## Image Captions Complete

- [ ] Each image that needs context has a caption.
- [ ] Captions identify the object, place, process, artifact, or observation.
- [ ] Captions explain why the image matters.
- [ ] Dates, locations, credits, and uncertainty are included where relevant.
- [ ] Captions do not add unapproved story details.

## Alt Text Prepared

- [ ] Informative images have meaningful alt text.
- [ ] Decorative images, if any, have empty alt text.
- [ ] Artifact alt text identifies the object and relevant visible detail.
- [ ] Field Note alt text explains the observation or evidence the image provides.
- [ ] Alt text does not duplicate long captions or carry unsupported claims.

## Metadata Prepared

- [ ] Content type is identified.
- [ ] Title is final or publication-approved.
- [ ] Excerpt or short description is prepared.
- [ ] Date or date uncertainty is recorded where relevant.
- [ ] People, places, objects, categories, tags, credits, and status notes are recorded where relevant.
- [ ] Privacy, permission, copyright, or uncertainty notes are preserved where needed.

## Cross-Links Verified

- [ ] Links point to real approved destinations.
- [ ] Link text clearly describes the destination.
- [ ] Related items are linked only when the relationship is meaningful.
- [ ] No weak links were added just to make the page look complete.
- [ ] External links, if any, point to appropriate sources and are checked.

## Related Content Verified

- [ ] Related Projects are real and useful.
- [ ] Related Field Notes are real and useful.
- [ ] Related Workshop Journal entries are real and useful.
- [ ] Related Cabinet items or artifacts are real and useful.
- [ ] "You may also enjoy..." suggestions are limited and meaningful.

## Accessibility Review

- [ ] The page has one clear `h1`.
- [ ] Heading order is logical.
- [ ] Links are descriptive.
- [ ] Images have appropriate alt text.
- [ ] Captions are associated with their images.
- [ ] Metadata is presented clearly.
- [ ] No information depends on color alone.
- [ ] Keyboard navigation works.
- [ ] Focus states remain visible.
- [ ] The page remains readable on mobile and at zoomed text sizes.

## Proofreading

- [ ] Spelling has been checked.
- [ ] Names have been checked.
- [ ] Dates and places have been checked.
- [ ] Captions match images.
- [ ] Alt text matches images.
- [ ] Links work.
- [ ] No Workbench-only notes are visible unless intentionally retained and clearly labeled.
- [ ] Tone remains calm, specific, and genuine.

## Final Publication Approval

- [ ] Final content has been approved.
- [ ] Images have been approved.
- [ ] Captions, alt text, metadata, and related links have been approved.
- [ ] Privacy, permission, and copyright concerns have been reviewed.
- [ ] The page or shelf has been previewed in context.
- [ ] A rollback path exists if theme or WordPress changes are involved.
- [ ] Publication sequence has been approved.

## Image Preparation Guidelines

Gather photographs before publication whenever images would clarify, preserve, or document the subject.

### Artifacts

Preferred photographs:

- Overall object view.
- Close detail showing marks, signatures, wear, labels, handwriting, or construction.
- Context photograph showing how the object is kept or used, if approved.
- Scale or relationship photograph when size or placement matters.
- Related-object photograph when two artifacts belong together.

### Field Notes

Preferred photographs:

- The observed subject.
- The setup, environment, or conditions surrounding the observation.
- Process or comparison images when they make the observation clearer.
- Detail photographs that show what changed, succeeded, failed, or raised the question.
- Supporting notes, labels, sketches, or tools when they function as evidence.

### Projects

Preferred photographs:

- Project overview.
- Process stages.
- Tools, materials, or workspace context.
- Before/after or progress comparisons when useful.
- Details that explain decisions, repairs, failures, or lessons learned.

### Exhibits

Preferred photographs:

- Establishing image that identifies the exhibit subject.
- Evidence images that support the story.
- Detail images that reward close looking.
- Related object, place, process, or source images.
- Final context image showing why the exhibit belongs in Two-Bit Alchemy.

## Use Rule

If an image does not clarify, preserve, document, or deepen the content, it should wait.
