# Implementation Approval Checklist

Development should wait until these decisions are approved.

## Required Before Website Code

- Review `docs/brand-manifest.md` and confirm the proposed implementation respects it.
- Confirm platform direction.
- Confirm whether the existing WordPress site remains the production base.
- Complete or schedule WordPress backup and update process.
- Confirm initial launch scope.
- Confirm brand asset handling.
- Confirm typography and color exploration process.
- Confirm content migration approach.

## First Implementation Milestone Candidate

Recommended first milestone after approval:

1. Audit the existing WordPress site.
2. Record theme, plugin, content, performance, accessibility, and security findings.
3. Recommend a narrow implementation path based on the audit.
4. Build only the smallest approved template or prototype needed to validate direction.
