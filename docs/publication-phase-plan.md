It is planning documentation only. It does not publish content, modify the theme, add templates, redesign pages, change CSS, or change JavaScript.

### 2. Page And Template Requirements

Do not add new templates for this phase unless a later approval specifically changes that constraint.

Objective requirements:

- Use the existing public framework.
- Use the existing Field Notes page or native post/page structure for Black Willow.
- Use the existing Cabinet page for artifact introduction and shelf context unless a later approved content model requires individual artifact pages.
- Do not redesign the homepage, Field Notes page, Cabinet page, or artifact presentation.
- Do not add decorative artwork, animation, or unrelated layout changes.
- Do not place personal story content only inside PHP templates if it can reasonably live as repository content.

Implementation choices requiring approval before build:

### 3. Image Preparation Workflow

Images are evidence, not decoration.

Required workflow:

1. Capture original photographs.
2. Store originals outside the theme until selected for publication.
3. Select only images that clarify, preserve, document, or deepen the shelf content.
4. Create web-ready exports with descriptive filenames.
5. Preserve original files separately from optimized exports.
6. Record photographer, date if known, object shown, and any credit or permission note.
7. Optimize images before upload.
8. Verify crops do not hide meaningful context.
9. Confirm images are not misleading, over-restored, or presented as evidence for claims they do not support.

Likely image set:

- Black willow cuttings or trees.
- Rooting water or propagation setup, if available.
- Lavender cutting result, if available.
- Signed Tori Amos Camel cigarette pack.
- Shelf context for the signed cigarette pack, if approved.
- White 1981 Ford Escort model in display case.
- Ford Escort model serving as pedestal for the signed cigarette pack, if approved.

### 4. Caption Requirements

Every published image should have a caption when the image needs context.

Caption checklist:

- Identify the object, plant, setup, or scene.
- Include date or approximate date only when known or clearly marked uncertain.
- Explain why the image matters to the shelf item.
- Include source, credit, or permission note where needed.
- Avoid merely repeating visible information.
- Avoid adding story details that are not approved elsewhere.

Artifact captions should explain the artifact's relationship to the story.

Field Note captions should explain what the image helps the visitor observe.

### 7. Accessibility

Minimum accessibility requirements:

- One clear `h1` per public page.
- Logical heading order.
- Descriptive link text.
- Informative images have meaningful alt text.
- Decorative images, if any, use empty alt text.
- Figures use captions when context is needed.
- Metadata is presented in semantic lists or clear labeled groups.
- No information is conveyed by image alone.
- Related content regions are clearly labeled.
- Keyboard navigation remains usable.
- Text remains readable at mobile widths and zoomed views.

Alt text requirements:

- Artifact alt text should identify the object and the relevant visible detail.
- Field Note alt text should describe what the visitor needs to understand from the image.
- Alt text should not carry unapproved story details or duplicate long captions.

### 9. Quality-Control Review

Editorial review:

- Confirm each item answers the five Exhibit Standard questions.
- Confirm no placeholders remain unless intentionally public and clearly labeled.
- Confirm no invented facts were added.
- Confirm uncertain details remain honest.
- Confirm the tone matches the Voice and Editorial Guide.
- Confirm private or sensitive details have been reviewed.

Content review:

- Check spelling of names and places.
- Check date ranges and uncertainty labels.
- Confirm image captions and alt text match the images.
- Confirm metadata is factual.
- Confirm related links work and point to approved destinations.

Technical review:

- Preview desktop and mobile.
- Check keyboard navigation.
- Check focus states.
- Check page headings.
- Check image sizes and loading behavior.
- Check browser console.
- Check WordPress editor/admin screens if content is entered manually.

Publication review:

- Confirm backup exists before live changes.
- Confirm previous working theme package is available for rollback.
- Confirm updated theme package, if needed, installs correctly.
- Confirm public pages are reachable from navigation or approved cross-links.
- Confirm no Workbench paths or unpublished notes are exposed publicly.

### 10. Publication Sequence

Recommended sequence:

1. Complete repository publication drafts for all three shelf items.
2. Review shelf worthiness and confirm all three items belong in the first public shelf.
3. Complete fact review, privacy review, and copyright/permission review.
4. Prepare and optimize images.
5. Write captions, alt text, credits, and metadata.
6. Decide final URL and placement for each item.
7. Update repository content files.
8. Implement only approved public rendering changes.
9. Preview locally or in staging.
10. Run accessibility and quality-control checks.
11. Package the theme if theme changes are required.
12. Back up WordPress before deployment.
13. Deploy theme/content updates.
14. Verify public pages after deployment.
15. Update the implementation log with what was published.

- Add captions, alt text, metadata, and credits before publication.
- Keep uncertainty labels where facts are incomplete.

## Ready-To-Publish Criteria

The first shelf is ready to publish only when:

- All three shelf items have approved final content.
- All public content exists in repository source files or approved WordPress content.
- Images, captions, alt text, metadata, credits, and permission notes are complete.
- Sensitive personal details have been reviewed.
- Copyright-sensitive material has been reviewed.
- Related links are meaningful and working.
- The public pages have been previewed on desktop and mobile.
- Accessibility checks pass at the level required by the Exhibit Standard.
- The final publication sequence has been approved.
